Задание 1

Инициализация нового проекта и создание моделей с ассоциацией One-to-Many.

    1. Инициализация нового проекта

Создайте папку для вашего проекта.
Откройте терминал и перейдите в директорию, где вы хотите создать проект с помощью команды `cd project_name`.

    2. Инициализируйте новый проект Node.js:

Внутри папки вашего проекта выполните команду `npm init -y`. Это создаст файл `package.json`, который будет хранить информацию о вашем проекте и его зависимостях.
Не забудьте добавить `"type": "module"` в package.json.

    3. Установите необходимые зависимости:

Установите Express, Sequelize и MySQL2 с помощью команды:
npm install express sequelize mysql2

Также установите пакет `sequelize-cli`, который поможет вам в создании моделей и миграций:
npm install --save-dev sequelize-cli

    4. Инициализируйте Sequelize:

Выполните команду `npx sequelize-cli init`, чтобы создать базовую структуру для работы с Sequelize. Это создаст папки `config`, `models`, `migrations` и `seeders`.

    5. Настройте файл конфигурации:

В папке `config` откройте файл `config.json`.
В этом файле вы найдете конфигурации для различных окружений (development, test, production). Убедитесь, что все данные для подключения к вашей базе данных MySQL указаны правильно (имя базы данных, пользователь, пароль, хост и диалект).

    6. Создайте файл для подключения к базе данных:

Создайте файл `db.js` в папке `config`.
В этом файле настроите подключение к базе данных, используя параметры из `config.json`.

    7. Создайте две новые модели:

Перейдите в папку `models`.
Создайте файлы моделей `Category` и `Product`.

    8. Настройте две новые модели:

Настройте модель `Category` так, чтобы у нее было поле `name`.
Настройте модель `Product` так, чтобы у нее были поля `name` и `price`.

    9. Настройте ассоциацию One-to-Many:

Откройте файл модели `Product`.
Добавьте зависимость в модель - внешний ключ `categoryName`, который указывает на категорию, связанную с этим товаром.

    10. Создайте миграции для моделей:

Сгенерируйте миграции для `Category` и `Product`:
npx sequelize-cli migration:generate --name create-categories
npx sequelize-cli migration:generate --name create-products

    11. Проверьте и при необходимости измените миграции:

Откройте файлы миграций в папке `migrations`. Убедитесь, что структура таблиц соответствует вашим ожиданиям и включает внешний ключ для связи One-to-Many.

    12. Настройте ассоциацию:

В папке `models` создайте файл `index.js`.
Внутри файла `index.js` импортируйте модели `Category` и `Product`.
Установите связь One-to-Many между таблицами.

    13. Выполните миграции:

В терминале выполните команду `npx sequelize-cli db:migrate`, чтобы применить миграции. Эта команда создаст таблицы в базе данных, используя данные из миграций.

    14. Проверьте структуру базы данных:

Откройте MySQL Workbench и подключитесь к вашей базе данных.
Перейдите в раздел, где отображаются таблицы вашей базы данных, и убедитесь, что обе таблицы (`Categories` и `Products`) созданы корректно.

    15. Проверьте, что связь One-to-Many настроена правильно:

Убедитесь, что в таблице `Products` есть поле, которое выступает в роли внешнего ключа, указывающего на таблицу `Categories`.

	Задание 2

Проверка ассоциации One-to-Many.

    1. Создайте новый файл для скрипта:

В корневой директории вашего проекта создайте новый файл, назовите его, например, `one-to-many-script.js`. Этот файл будет использоваться для выполнения операций, связанных с проверкой ассоциации One-to-Many.

    2. Импортируйте необходимые модули:

В новом файле нужно будет импортировать модули для работы с базой данных и вашими моделями. Убедитесь, что вы правильно указываете пути к файлам и модулям.

    3. Установите соединение с базой данных:

В начале скрипта настройте соединение с вашей базой данных, используя `sequelize.authenticate()`.
Это поможет убедиться, что соединение с базой данных установлено успешно перед выполнением операций.

    *4. Синхронизируйте модели с базой данных:

Используйте метод `sequelize.sync({ force: true })`, чтобы пересоздать таблицы в базе данных.
Параметр `{ force: true }` гарантирует, что таблицы будут пересозданы заново, удалив существующие данные.
Это важно для тестирования, чтобы всегда начинать с чистой базы.

    5. Создайте новую запись для категории:

В скрипте создайте новую категорию, используя метод `create()` для модели `Category`. Убедитесь, что добавляете необходимые данные - название категории.

    6. Создайте записи для продуктов:

Далее, создайте несколько продуктов, связанных с этой категорией, используя метод `create()` для модели `Product`. При создании продукта укажите категорию, к которой он должен принадлежать.

    7. Загрузите категорию с продуктами:

В скрипте выполните запрос для загрузки категории вместе со связанными продуктами. Используйте метод `findOne()` для поиска категории, и включите связанные продукты, используя опцию `include`.

    8. Проверьте и выведите результаты:

После выполнения запроса выведите результаты в консоль. Это покажет, что продукты успешно связаны с категорией, и ассоциация One-to-Many работает правильно.

    9. Добавьте обработку ошибок:

Оберните ваш код в `try...catch`, чтобы корректно обрабатывать возможные ошибки при выполнении операций.

    10. Закройте соединение с базой данных:

В `finally` блоке закройте соединение с базой данных, чтобы освободить ресурсы после завершения работы скрипта.

    11. Запустите скрипт:

Откройте терминал в корневой директории проекта и выполните команду `node one-to-many-script.js`, чтобы запустить ваш скрипт.
Проверьте вывод в консоли, чтобы убедиться, что ассоциация работает правильно, и данные связаны корректно.

    12. Проверьте результаты в MySQL Workbench:

Подключитесь к вашей базе данных через MySQL Workbench и проверьте таблицы `Categories` и `Products`. Убедитесь, что данные созданы правильно, и что внешние ключи установлены корректно.
