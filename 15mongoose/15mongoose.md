Задание 1

Создание и настройка проекта

    1. Инициализация нового проекта

Откройте терминал и перейдите в папку, где вы хотите создать проект.
Введите команду `npm init -y`, чтобы создать файл `package.json` с настройками вашего проекта.

    2. Установка необходимых зависимостей

Установите Express, Mongoose, dotenv

    3. Создание файла `app.js` для запуска Express-сервера

Создайте в корневой папке проекта файл с именем `app.js`.
В этом файле настройте ваш Express-сервер.
Вам нужно создать основной сервер Express и настроить его на прослушивание порта 3000.

    4. Настройка подключения к базе данных MongoDB

Создайте файл `.env` в корневой папке вашего проекта. Этот файл будет использоваться для хранения конфиденциальных данных, таких как URL для подключения к MongoDB.
Внутри файла `.env` добавьте строку с переменной, которая будет хранить URL вашей базы данных MongoDB.
В файле `app.js` используйте библиотеку `dotenv`, чтобы загрузить переменные окружения из файла `.env`. Это позволит вашему приложению подключаться к базе данных, используя защищенный URL.
Настройте Mongoose для подключения к MongoDB с использованием переменной из файла `.env`. Убедитесь, что вы добавили код для обработки ошибок подключения.

    5. Запуск сервера и проверка подключения

Убедитесь, что сервер успешно запущен и подключение к базе данных установлено. Если вы видите сообщения об успешном подключении в терминале, значит все настроено правильно.

    Задание 2

Операция Create (Создание документа)

    1. Создание схемы и модели

В папке `models` создайте файл для новой модели, например, `Product.js`.
Определите схему для продукта (Product) с полями `name`, `price`, и `category`. Настройте типы данных и валидацию (например, `name` должен быть обязательным, а `price` должен быть числом).

    2. Настройка маршрута для создания продукта

В файле `app.js` подключите модель `Product`.
Создайте маршрут `POST /products`, который будет принимать данные нового продукта и сохранять их в базе данных, используя модель `Product`.

    3. Тестирование создания продукта

Отправьте POST-запрос на маршрут `POST /products` с данными нового продукта.
Проверьте, что продукт успешно создан и данные корректно сохраняются в базе данных.

    Задание 3

Операция Read (Чтение данных)

    1. Настройка маршрута для чтения продуктов

В файле `app.js` создайте маршрут `GET /products`, который будет возвращать список всех продуктов из базы данных.
Реализуйте фильтрацию по категории и сортировку по цене.

    2. Тестирование чтения данных

Отправьте GET-запрос на маршрут `GET /products` и убедитесь, что возвращается список всех продуктов с фильтрацией и сортировкой.

    Задание 4

Операция Update (Обновление данных)

    1. Настройка маршрута для обновления продукта

В файле `app.js` создайте маршрут `PUT /products/:id`, который будет обновлять данные продукта по его ID.
Используйте метод `findByIdAndUpdate()` для обновления документа в базе данных.

    2. Тестирование обновления продукта

Отправьте PUT-запрос на маршрут `PUT /products/:id` с новыми данными продукта.
Убедитесь, что данные успешно обновляются в базе данных.

    Задание 5

Операция Delete (Удаление данных)

    1. Настройка маршрута для удаления продукта

В файле `app.js` создайте маршрут `DELETE /products/:id`, который будет удалять продукт по его ID.
Используйте метод `findByIdAndDelete()` для удаления документа из базы данных.

    2. Тестирование удаления продукта

Отправьте DELETE-запрос на маршрут `DELETE /products/:id` и убедитесь, что продукт успешно удален из базы данных.
